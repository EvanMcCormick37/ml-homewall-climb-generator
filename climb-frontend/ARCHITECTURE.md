# Architecture — climb-frontend

React + TypeScript frontend for BetaZero. Lets users browse layouts, place holds on a canvas, and generate/save/browse climbing routes.

---

## Tech Stack

| Concern   | Library / Version                                       |
| --------- | ------------------------------------------------------- |
| Framework | React 19 + TypeScript 5.9                               |
| Build     | Vite 7.2 with `@vitejs/plugin-react`                    |
| Routing   | TanStack Router 1.143 (file-based, auto-generated tree) |
| Auth      | Clerk 5.61 (`@clerk/clerk-react`)                       |
| HTTP      | Axios 1.13 with Bearer token interceptor                |
| Styling   | Tailwind CSS 4 + CSS variables (BetaZero design system) |
| Icons     | lucide-react 0.562                                      |

## Route Structure

File-based routing (`src/routes/`). TanStack Router auto-generates `routeTree.gen.ts` at build time.

| File                    | Route              | Purpose                                    |
| ----------------------- | ------------------ | ------------------------------------------ |
| `__root.tsx`            | (root)             | ClerkProvider + auth token init            |
| `index.tsx`             | `/`                | Homepage — layout cards, hero, "Add Wall"  |
| `signIn.tsx`            | `/signIn`          | Clerk hosted sign-in redirect              |
| `signUp.tsx`            | `/signUp`          | Clerk hosted sign-up redirect              |
| `layouts/new.tsx`       | `/layouts/new`     | 4-step layout creation wizard              |
| `$layoutId/view.tsx`    | `/$layoutId/view`  | Saved climb browser (filters, sort, canvas)|
| `$layoutId/set.tsx`     | `/$layoutId/set`   | Climb generator page                       |
| `$layoutId/holds.tsx`   | `/$layoutId/holds` | Interactive hold placement editor          |
| `$layoutId/sizes.tsx`   | `/$layoutId/sizes` | Size variant management                    |

### Page Summaries

**Home (`/`)** — Hero section, scrolling layout grid (photo, name, dimensions, climb count), "Add Your Wall" button (signed-in only), nav links.

**Create Layout (`/layouts/new`)** — 4-step wizard:
1. Choose visibility (`public` / `unlisted` / `private`)
2. Upload photo (drag-drop or file picker)
3. Crop: rectangular (`ImageCropper`) or trapezoid (`TrapezoidCropper`) for perspective correction
4. Details: name, dimensions (ft), default angle, kickboard flag

On submit: `POST /layouts` → `PUT /layouts/{id}/photo` → `POST /layouts/{id}/sizes` → redirect to `/$layoutId/holds`.

**Holds Editor (`/$layoutId/holds`)** — Interactive canvas for placing and editing holds. Modes: `add` / `remove` / `select` / `edit`. On save: `PUT /layouts/{id}/holds` replaces the full hold array atomically.

**Climb Generator (`/$layoutId/set`)** — User selects grade, angle, generation settings (timesteps, guidance, etc.); calls `GET /layouts/{id}/generate`; navigates generated results; saves chosen climbs via `POST /layouts/{id}/climbs`.

**Climb Viewer (`/$layoutId/view`)** — Filters (grade range, angle, setter, tags, holds), sort options (date, ascents, grade, quality), climb visualization on the wall canvas, delete (owner/setter only), share-link copy.

**Size Manager (`/$layoutId/sizes`)** — Lists sizes; create new size (name, edge bounds, kickboard); delete sizes.

## API Client

`src/api/client.ts` — Axios instance pointed at `VITE_API_URL`. A request interceptor calls `getAuthToken()` and injects the result as `Authorization: Bearer {token}` before every request.

`setAuthTokenProvider(fn)` is called from `__root.tsx` after Clerk initializes, wiring in `getToken()` from Clerk's `useAuth()` hook.

### API Functions

**`src/api/layouts.ts`**

| Function                          | HTTP Call                                    |
| --------------------------------- | -------------------------------------------- |
| `getLayouts()`                    | `GET /layouts`                               |
| `getLayout(id, sizeId?)`          | `GET /layouts/{id}[?size_id=]`               |
| `createLayout(data)`              | `POST /layouts` (FormData)                   |
| `updateLayout(id, data)`          | `PUT /layouts/{id}/edit`                     |
| `uploadLayoutPhoto(id, file)`     | `PUT /layouts/{id}/photo` (multipart)        |
| `deleteLayout(id)`                | `DELETE /layouts/{id}`                       |
| `setLayoutHolds(id, holds)`       | `PUT /layouts/{id}/holds`                    |
| `fetchLayoutPhoto(id)`            | `GET /layouts/{id}/photo` → blob URL         |
| `fetchLayoutPhotoSmall(id)`       | `GET /layouts/{id}/photo-small` → blob URL   |
| `getLayoutPhotoUrl(id)`           | Returns a static URL string (no auth fetch)  |

**`src/api/sizes.ts`**

| Function                      | HTTP Call                                |
| ----------------------------- | ---------------------------------------- |
| `getSizes(layoutId)`          | `GET /layouts/{id}/sizes`                |

**`src/api/climbs.ts`**

**`src/api/generate.ts`**

## Hooks

| Hook                      | File                    | Returns / Purpose                                                     |
| ------------------------- | ----------------------- | --------------------------------------------------------------------- |
| `useBetaZeroAuth()`       | `hooks/useAuth.ts`      | `{ user, isLoaded, isSignedIn, userId, displayName, email, getApiToken }` |
| `useLayouts()`            | `hooks/useLayouts.ts`   | `{ layouts, loading, waking, error, refetch }`                        |
| `useLayout(id, sizeId?)`  | `hooks/useLayouts.ts`   | `{ layout, loading, waking, error, refetch }`                         |
| `useClimbs(layoutId, filters)` | `hooks/useClimbs.ts` | `{ climbs, total, loading, error, filters, setFilters, refetch, removeClimb, selectedClimb, setSelectedClimb }` |
| `useHolds(layoutId)`      | `hooks/useHolds.ts`     | Hold state + pixel↔feet coordinate conversion                         |
| `useImageCrop()`          | `hooks/useImageCrop.ts` | `{ imageUrl, cropArea, isDragging, setImage, startDrag, … }`          |

`fetchWithWakeRetry()` (defined in `useLayouts.ts`) retries 502 responses for up to 20 s and exposes a `waking` boolean — handling cold starts on free-tier hosting. Both `useLayouts` and `useLayout` use this.

## Coordinate System

Holds are stored in **feet** (origin: bottom-left of wall). The canvas displays them in **pixels** (origin: top-left of image). `useHolds` converts between the two:

```
toFeetCoords:   x = pixelX / imgWidth  * wallWidth
                y = (imgHeight - pixelY) / imgHeight * wallHeight   ← Y is inverted

toPixelCoords:  px = hold.x / wallWidth  * imgWidth
                py = imgHeight - (hold.y / wallHeight * imgHeight)
```

## Authentication

**Provider**: Clerk.

1. `__root.tsx` wraps the app in `<ClerkProvider>` and calls `setAuthTokenProvider(() => getToken())` after Clerk initializes.
2. The Axios interceptor calls the provider before every request and injects `Authorization: Bearer {token}`.
3. Route loaders fall back to `window.Clerk?.session?.getToken()` when the hook-based provider isn't available.
4. Auth-gated pages check `isLoaded` then `isSignedIn` (from `useUser()`) and redirect to `/signIn` if not authenticated.
